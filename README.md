# API
api自动化测试
解决问题：
1.测试接口编写是否符合规范
2.测试数据格式检查是否完整
3.请求数据是否正确处理
    1>数据完整性
    2>类型是否准确
    3>上下文数据交互
4.定位错误原因
5.清除脏数据
6.清晰的测试报告

结构：
core.bash_url下编写接口url
core下按模块编写接口基本信息
tests下按功能模块编写测试用例
utils下包含常用的数据生成方法，测试用例转化工具
config:测试环境，账号信息，日志等的配置文件
pytest.ini: 测试用例的执行范围配置
run:测试用例的执行

用例编写：
采用allure + pytest
allure标记用例的步骤和优先级
pytest编写测试用例的步骤和参数化数据
测试数据可以采用yaml或excel
